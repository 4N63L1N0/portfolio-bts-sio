Homelab Docker sécurisé: reverse proxy, SSO et exposition maîtrisée des services

Professionnalisation d'un homelab personnel en plateforme d'hébergement orientée cybersécurité, avec contrôle d'accès centralisé, segmentation logique et réduction de la surface d'attaque.

Discipline: Cyber / Infrastructure
Statut: Réalisé / Améliorable
Technos: Docker, Docker Compose, Nginx Proxy Manager, Authelia, DNS local

Contexte 🧭

Homelab Docker professionnalisé pour héberger plusieurs services (dashboard, monitoring, outils admin) avec une approche cybersécurité: point d'entrée unique, authentification centralisée et exposition contrôlée.

Mon rôle 🛠️

  • 🧱 Concevoir l'architecture et les flux d'accès.
  • 🌐 Déployer Nginx Proxy Manager (reverse proxy + TLS).
  • 🔐 Intégrer Authelia pour protéger les services sensibles.

Architecture 🏗️

  • 🚪 Entrée unique via Nginx Proxy Manager (80/443).
  • 🛡️ Authentification en frontal via Authelia (MFA possible).
  • 📦 Services exposés uniquement derrière le reverse proxy.
  • 🔀 Isolation via réseaux Docker dédiés.

Analyse cybersécurité 🔎

  • ⚠️ Surface d'attaque: proxy web, interfaces admin, secrets applicatifs, accès distant.
  • ✅ Protections: contrôle d'accès centralisé, segmentation, moindre privilège.
  • 🚨 Risques résiduels: erreur de configuration.

Bonnes pratiques appliquées 📋

  • 🎯 Point d'entrée unique pour tous les services web.
  • 🔐 Authentification avant exposition des applications.
  • 🔒 Normalisation des accès.

Compétences démontrées 🎯

  • 🛡️ Architecture sécurisée en environnement conteneurisé.
  • 🌐 Reverse proxy, MFA.
  • 📊 Analyse de risques orientée exploitation réelle.

Améliorations possibles 🚀

  • 📡 Ajouter SIEM (Wazuh/Elastic) et alerting sécurité.
  • 🧪 Authentification en frontal a double facteur (MFA possible)
  • 🔑 Externaliser les secrets (Vault/SOPS) et automatiser (Ansible).
